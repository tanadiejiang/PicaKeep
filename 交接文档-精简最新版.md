## 项目定位

PicaKeep 是一个以本地漫画/本地图集/已下载管理为主线，同时支持客户端/服务端双模式与浏览器网页后台的 Flutter 项目。当前版本已稳定运行，`flutter analyze` 0 issues，`flutter build apk --debug` 通过。

## 模块功能清单

### 远程服务与连接池

- 远程图片并发加载：阅读闸（默认 6）/ 浏览闸（默认 8），可在设置-阅读/浏览中 1~12 实时调节
- 连接池分流：图片 bulk 池（24 连接）与 JSON 控制池（6 连接）隔离，封面走 bulk 池
- 连接池泄漏保护：孤儿连接超时回收 + 非 2xx 响应体 drain + body idle timeout 守卫 ⚠️ **修改图片/控制面请求链路必须读 [经验版 §1](./交接文档-经验思路版.md#1-远程服务与连接池)**
- 封面磁盘缓存：带失败负缓存（404 冷却 10 分钟），自动按大小限制裁剪旧文件

### 收藏 / 图片收藏远程档

- 收藏页远程档：列出远程收藏文件夹，支持中文路径，点击进阅读
- 图片收藏远程档：网格展示缩略图，无图项静默破图占位，点击定位资源进阅读
- 远程可用性判断、档位开关无条件显示 ⚠️ **修改收藏/图片收藏远程链路必须读 [经验版 §2](./交接文档-经验思路版.md#2-收藏图片收藏远程档)**

### 加密压缩包

- 加密 ZIP 真实密码验证：AES-128/192/256 + ZipCrypto 双模式，错密码不再假成功

### 服务端模式

- 运行模式切换：客户端/服务端，入口在 APP能力 页
- 服务信息页：状态摘要、mDNS / Bonjour 发现、网段扫描、刷新状态（真实触发重扫）、断开连接
- 服务信息页稳定性：隐藏加载圈空闲时不保留动画；服务统计变化只静默刷新统计快照，不触发全页 loading，跳动问题与 mDNS / 网段扫描模式无直接关系
- 局域网发现策略：`mDNS / Bonjour` 是真实零配置发现，`网段扫描` 保留旧 `/status` 探测；两处配置入口（服务信息页、APP能力页）共用同一组挡位与“mDNS 失败自动网段扫描”开关
- mDNS / Bonjour 发现边界：自研 UDP 5353 发布/发现，Windows 服务端需重点关注多网卡组播加入、A 记录地址与防火墙；Linux/Android 能互扫但扫不到 Windows 时，优先查 `picakeep_mdns.dart` 的网卡级组播/响应路径，不要退化成把网段扫描伪装成 mDNS
- 网段扫描行为：继续使用 `/status` 探测作为独立“网段扫描”档位；当前已连接地址会优先探测，但命中后仍继续扫完整候选网段并聚合多个服务，不再因为第一批命中而提前停止
- 多设备 / 多服务发现：最终候选 2+ 时服务信息页弹选择窗口，展示来源、设备/服务名、地址端口、漫画数量、延迟和服务描述；同地址去重，同设备不同端口保留
- Android mDNS：已补 `CHANGE_WIFI_MULTICAST_STATE` + multicast lock；UDP 5353 绑定在 Android/Windows 不启用 `reusePort`，避免 `reusePort not supported`
- 远程资源热同步：librarySignature + WebSocket 事件推送，阅读期间延迟刷新
- 服务端挡位热更新：host/port 不变时运行态切换，无需重启
- 远程已下载/资源库/图集：三档展示与切换，回前台/重新进入时强制刷新
- 服务端中文路径修复、favorites handler 优化、_notifyLibraryChanged 收口
- 收藏夹接口增强：列文件夹内容时按 candidates 匹配资源库补 `itemId` 等字段，cover 路由本地失效时回退资源库封面（`_favoriteCoverFallbackCache`）
- 远程封面路径优先级：cover.jpg > 首张图片，pageSizes 支持图片预占位 ⚠️ **修改服务端路由/扫描/同步必须读 [经验版 §4](./交接文档-经验思路版.md#4-服务端模式)**
