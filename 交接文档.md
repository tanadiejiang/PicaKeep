# PicaKeep 交接文档

## 1. 项目定位

`PicaKeep` 当前已经不是单纯的本地离线阅读器裁剪版，而是一个以本地漫画 / 本地图集 / 已下载管理为主线，同时开始落地客户端 / 服务端双模式的 Flutter 项目。

当前工作区中同时存在两个相关工程：
- `D:\Flutter_Projucts\PicaComic\PicaKeep`：当前主要维护目标。
- `D:\Flutter_Projucts\PicaComic\PicaComic`：原项目，对照 UI、阅读器行为、旧数据结构与兼容读取链路时会用到。

当前版本的实际主线已经分成两层：
- **本地层**：本地漫画、本地图集、已下载、收藏、历史、阅读器、设置。
- **远程层**：运行模式、服务信息、局域网扫描、远程资源库 / 远程已下载 / 远程图集、服务端本地 HTTP 与后台原型。

- 之前遇到的“受限 `download.db` 读不到就误判成新库，最后恢复只剩一条”的问题已修正：现在会先区分“不可读”和“真的不存在”，避免把原库误建成空库
- 普通删除、Root 删除与 Shizuku 删除现在都沿同一套受限路径判断走，权限不足时直接提示，不再装作成功
- 阅读设置页的 `keepScreenOn` 平台通道已补齐，`MissingPluginException` 的常亮报错已修掉
